# Feature Roadmap

## Phase A — Strong manager demo

Use fake/demo data or local sample data.

Features:

- Dashboard shell from current HTML.
- Morning brief.
- Today's Radar.
- AI Analysis panel.
- Manager Memory panel.
- Demo focus mode interaction.
- Demo draft approval.

Goal:

> Show the manager exactly what the product will feel like.

## Phase B — MVP with real Outlook

Features:

- Supabase auth.
- Microsoft Outlook connect.
- Initial Inbox and Sent Items sync.
- Unified `work_items`.
- Follow-up detection.
- Waiting-on-manager detection.
- Rule-based priority scoring.
- AI analysis with structured output.
- Dashboard with real data.

Goal:

> The manager can open Vesta and see what needs attention today.

## Phase C — Manager actions

Features:

- Manual task creation.
- Natural language task parser.
- Reminders.
- Snooze/done.
- Draft reply generation.
- Edit and approve send.
- Audit log.

Goal:

> The manager can clear work from the dashboard.

## Phase D — Memory and behavior learning

Features:

- VIP people.
- Delegation rules.
- Tone preferences.
- Do-not-do rules.
- Memory review.
- AI-suggested memories requiring approval.
- Memory retrieval for analysis and drafts.

Goal:

> The assistant starts behaving like it understands the manager.

## Phase E — New manager-dream features

Features:

- AI Decision Desk.
- Promise and Commitment Tracker.
- Clear My Day / Focus Mode.
- Relationship Risk Heatmap.
- Meeting Prep Pack after calendar integration.
- Weekly Executive Review.

Goal:

> Vesta becomes a manager operating system, not only an inbox assistant.

## Phase F — Teams and company-wide expansion

Start simple:

1. Teams notifications for briefs/reminders.
2. Teams bot mentions.
3. Selected chats/channels.
4. Wider Teams monitoring only after admin/security review.

Company-wide later:

- Organizations.
- Manager roles.
- Tenant onboarding.
- Admin consent.
- Department-level blocker analytics.
- Privacy-safe executive situation room.
